Google Drive Setup

Google Client ID

Paste the Web OAuth Client ID from your Google Cloud project, then Save and click Connect Drive. A Client ID is not a secret, so it is safe to keep here.

One-time setup steps

1. Host this HTML on your GitHub Pages so it has an https web address.
2. Go to Google Cloud Console, create or pick a project.
3. APIs & Services → Library → enable Google Drive API.
4. OAuth consent screen: type External; add your own Google account under Test users.
5. Credentials → Create Credentials → OAuth client ID → type Web application.
6. Under Authorized JavaScript origins add your GitHub Pages origin only, for example https://yourname.github.io (no path, no trailing slash).
7. Copy the generated Client ID into the box above and Save.
The app uses the drive.file scope, so it can only see the single diary file it creates, nothing else in your Drive.
